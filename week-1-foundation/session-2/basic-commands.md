#Documentation of Initialization Command, Right Commit Message, Command to Check Commit History
# and a Personal Frequently Used Command in Git and CLI

#Repository Initialization
** Command** - git init (Creates a hidden .git folder where all commits will be saved.

#Right Commit Messages
To write a good commit message, you'd have to state the type of commit you're making followed
by a colon and then the actual commit message which shouldn't be more than 72 characters. Examples 
of such terms include: docs(mostly used), chore, feat, refactor. Avoid using terms like update,
new etc.

#Command to Check All Commits
* **git log** - Shows all commits in full
* **git log -oneline** - Shows all commits on a single
