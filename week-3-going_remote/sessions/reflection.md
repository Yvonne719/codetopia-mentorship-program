● Why do remote repositories matter?
● What does the -u flag do?
● How do local and remote repositories stay synchronized?
● What challenges did you face?
● What did you learn this week?


#ANSWERS
- Remote repositories matter because they keep a copy of our project in the cloud. Such projects 
hardly lost. They serve as an active resume. Employers can check them with just a link to them. It
also makes collaboration on a project easier.

- The -u flag creates a tracking link between your local branch and a branch in your remote repository.

- Local and remote repositories stay synchronized with the continuous use of git pull and git push
whenever a change is made in your local or remote repository respectively.

- I made a change on GitHub and forgot to pull before pushing a new change. I got an error because 
of this. I also had a challenge with the deliverables that asked to create a new repo. My repository
tree is kind of long because of that.

- This week, I understood the beauty of how my local repo can be linked to my remote repo and how to
properly and easily sync changes between both.
